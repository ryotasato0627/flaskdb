# FlaskDB

## Overview
このプロジェクトは、個人用ノートを管理するためのREST APIです。  
ユーザーはノートの作成・取得・更新・削除（CRUD）を行うことができます。

Python（Flask）を用いて実装しており、バックエンド開発の基礎となる  
API設計、責務分離、テストの書き方を学ぶことを目的とした学習用プロジェクトです。

---

## Purpose
- 現職での研修に加えて、フレームワークやDB連携について
  自主的に理解を深める必要があると考え、本プロジェクトを開始しました。
- 実務に近い形でバックエンドAPIの設計・実装を経験することを目的としています。

---
## Design Policy
本プロジェクトでは、以下の点を設計方針として重視しました。

- **責務の分離**  
  コントローラーはHTTPの入出力に専念し、業務ロジックはService層に集約しています。

- **明示的なバリデーションとエラーハンドリング**  
  不正な入力はService層で検証し、想定外の動作を避けるようにしています。

- **テストしやすい構成**  
  依存関係を分離し、単体テストで業務ロジックを検証できるようにしました。

---

## Architecture
```
app/
 ├─ config.py
 ├─ database.py
 ├─ models/
 ├─ repository/
 ├─ routes/
 ├─ schemas/
 ├─ services/
 └─ utils/
```
この構成により、各レイヤーの責務を明確にし、
コードの可読性と保守性を高めることを意図しています。

---

## Testing Strategy
テストは主にService層を対象とした単体テストを実施しています。

- pytestを使用
- Repository層はモック化し、業務ロジックを独立して検証
- 正常系および一部の異常系をテスト対象としています

なお、E2Eテストや結合テストについては、  
学習範囲を考慮し、本プロジェクトでは対象外としています。


---
## Trade-offs
学習プロジェクトとして、実装範囲を限定している部分があります。

- 認証・認可はアクセストークンまでにとどめ、個人利用のみを想定しています
- セキュリティやパフォーマンスの最適化は最低限にしています
